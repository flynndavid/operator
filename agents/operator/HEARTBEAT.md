# HEARTBEAT.md — Morning Briefing

_Deliver this when {{CLIENT_NAME}} starts their day. Keep it tight. Make it actionable._

---

## The Pulse — Financial Vitals
_Source: Accounting Hub (QBO/Xero)_

- Cash position and runway
- Revenue vs. target (MTD)
- Outstanding invoices (overdue flagged)
- Upcoming expenses / bills due
- Anomalies (unexpected charges, failed payments, unusual patterns)

> If accounting hub isn't connected, skip with: "Pulse: No accounting hub connected yet."

---

## The Pipeline — Sales Funnel
_Source: CRM / pipeline tracker_

- Active deals and stage changes since last briefing
- Follow-ups due today
- Stalled opportunities (no movement in X days, calibrated to archetype)
- New inbound leads or inquiries
- Win/loss since last briefing

> If no CRM connected, surface what you know from conversations and memory.

---

## The People — Customer Health
_Source: Communication logs, CRM, memory_

- Customer interaction cadence vs. archetype benchmark
- Customers going quiet (flag by name)
- Retention risks or churn signals
- Recent positive signals (referrals, expansions, praise)
- Key relationships needing attention

> The customer is always north star. Lead with relationships, not numbers.

---

## The Ops Queue — Tasks Due
_Source: Task tracker, memory, calendar_

- Tasks due today
- Overdue items (with age)
- Upcoming deadlines this week
- Blocked items needing founder input
- Delegated tasks awaiting completion

> Prioritize by impact, not volume. Three important things > twelve trivial ones.

---

## The Build Progress — Strategic Phase
_Source: Current Build Mode plan_

- Current strategic initiative and phase
- Progress since last check-in
- Next milestone and ETA
- Blockers or risks
- Whether pace is sustainable

> If no active Build Mode initiative, skip with: "Build: No active strategic initiative. Want to start one?"

---

## Wellbeing Check

- Founder's recent pace and energy (observed from interaction patterns)
- Hours since last break / days since last day off (if trackable)
- Gentle nudge if burnout signals are present
- Acknowledge wins — even small ones

> This is not optional. The founder is a human being first. One line is enough: "You've been grinding hard this week. Solid progress — make sure you take a real break this weekend."

---

## Custom Tool Status
_Source: TOOLS.md, integration checks_

- Integration health (connected / disconnected / errors)
- Any tools needing attention or re-authentication
- Custom tools built: quick status on each

> Only include if there are tools to report on. Don't pad with "all systems normal" unless something was recently broken.

---

## Briefing Behavior

**Learn what matters.** Track which sections the founder engages with, asks follow-up questions about, or skips. Over time, weight the briefing toward what they care about and compress what they don't.

**Don't manufacture urgency.** If it's a quiet day, say so: "Clean slate today. Pipeline steady, no fires. Good day to work on [current Build initiative]."

**Be actionable.** Every item should either inform a decision or prompt a specific action. "Revenue is $X" is reporting. "Revenue is $X, which puts us 15% behind target — want to look at the pipeline together?" is useful.

**Adapt cadence.** Some founders want this daily. Some want it Monday/Wednesday/Friday. Some want it only when something changes. Ask during onboarding, then adjust based on behavior.
