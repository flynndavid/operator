# Vendor Management

## When to Use
When tracking vendors, managing contracts, scheduling follow-ups, or coordinating vendor logistics for events.

## What This Skill Does
Maintains a vendor database, tracks contract status, automates follow-up scheduling, and manages vendor relationships across events.

## How to Use

### Vendor Database
Maintain vendor info in `tasks/active/vendors.md`:

```markdown
# Vendor Directory

## Active Vendors

### [Vendor Name]
- **Category:** (Audio/Visual, Catering, Security, etc.)
- **Contact:** Name, email, phone
- **Contract status:** (Prospecting / Negotiating / Signed / Complete)
- **Contract value:** $X
- **Payment terms:** (Net 30, 50% upfront, etc.)
- **Payment status:** (Pending / Partial / Paid)
- **Performance rating:** ⭐⭐⭐⭐⭐ (from past events)
- **Notes:** 

## Prospective Vendors

### [Vendor Name]
- **Category:** 
- **Contact:** 
- **Source:** (Referral, search, etc.)
- **Status:** (Reached out / Meeting scheduled / Proposal received)
```

### Follow-up Automation
During heartbeats, check:
1. Vendors with proposals outstanding > 3 days → remind client to follow up
2. Contracts awaiting signature > 1 week → flag
3. Payments coming due in < 2 weeks → alert
4. Vendors not confirmed for upcoming event < 2 weeks out → escalate

### Vendor Communication Drafts
When drafting vendor communications:
- **Initial outreach:** Professional, specific about event details and what you need
- **Follow-ups:** Friendly but direct, reference previous conversation
- **Confirmations:** Clear on dates, times, load-in details, payment
- **Thank you:** Genuine, mention something specific they did well

### Best Practices (from the network)
- Send follow-ups Tuesday-Thursday mornings (highest response rate)
- Get written confirmation for everything — verbal agreements aren't enough
- Book preferred vendors 4-6 months out (they fill up fast)
- Always have a backup vendor for critical categories (AV, catering, security)
- Pay promptly — vendors talk to each other, and good payers get priority
- Ask for references and check them for new vendors
- Document vendor performance after each event for future decisions
